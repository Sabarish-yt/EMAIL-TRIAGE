## Overview
 
Clinical Triage is a real-world AI agent evaluation environment that simulates an Emergency Department (ED) triage workflow. Agents must demonstrate clinical reasoning across three tasks of increasing complexity:
 
1. **Vitals-Based Triage** (Easy) — Assign ESI triage levels from vital signs
2. **Diagnostic Workup Ordering** (Medium) — Select appropriate labs, imaging, and consults
3. **Differential Diagnosis & Disposition** (Hard) — Synthesize complex cases into diagnosis + admission decisions
 
The environment uses realistic patient presentations generated by emergency medicine domain experts, including edge cases, mimics, and high-acuity scenarios that challenge even frontier LLMs.
 
### Why This Environment Matters
 
ED triage errors are a leading cause of preventable patient harm. Training AI agents on structured clinical decision-making:
- Provides a high-stakes, real-world benchmark beyond toy tasks
- Creates agents that could assist triage nurses in under-resourced settings
- Tests multi-step reasoning, safety-critical caution, and cost efficiency simultaneously

## Setup & Usage
 
### Local Development
 
```bash
# Clone and install
git clone <repo-url>
cd clinical-triage-env
pip install -r requirements.txt
 
# Run tests
pytest tests/ -v
 
# Start the server
uvicorn app:app --host 0.0.0.0 --port 8000 --reload
 
# Open docs
open http://localhost:8000/docs
```


## Environment Details

### Observation Space
The observation space is a JSON object containing patient presentation:
- `patient_id` (str): Unique patient ID
- `age` (int): Patient age
- `sex` (str): Patient sex
- `chief_complaint` (str): Reason for ED visit
- `arrival_mode` (str): How the patient arrived (EMS, walk-in, etc.)
- `vitals` (dict): HR, BP, RR, Temp, SpO2, Pain, GCS, Weight
- `history` (dict): PMH, PSH, Meds, Allergies, Social
- `ros` (dict): Review of Systems
- `physical_exam` (dict): Provider findings
- `labs_ordered`, `imaging_ordered`, `consults_ordered` (list)
- `step_number` (int): Current step in trajectory

### Action Space
Agents must return an action matching the current task. Supported actions:
- `assign_triage_level`: (Task: easy_case) Requires `triage_level` (1-5)
- `order_workup`: (Task: medium_case) Requires list of `workup_items`
- `recommend_disposition`: (Task: hard_case) Requires primary_diagnosis, differential, disposition (discharge, admit_floor, admit_icu), and reasoning

### API Usage & Example Requests

#### 1. Reset Environment
```bash
curl -X POST http://localhost:8000/reset \
  -H "Content-Type: application/json" \
  -d '{"task_id": "easy_case", "seed": 42}'
```

#### 2. Take a Step
```bash
curl -X POST http://localhost:8000/step \
  -H "Content-Type: application/json" \
  -d '{
    "session_id": "<your-session-id>", 
    "action": {"action_type": "assign_triage_level", "triage_level": 1}
  }'
```

#### 3. View State
```bash
curl http://localhost:8000/state/<your-session-id>
```
